# Table Layout Agent

## Goal

重做牌桌主体，让它有清楚的四方位空间关系。桌面端应像真正的日麻牌桌：上家、下家、对家、自己位置明确；移动端则优先显示目标玩家，不强行塞完整四方桌。

## Ownership

负责：

- 牌桌主体组件结构和响应式布局。
- 桌面端四方位玩家区、中央局信息、目标玩家高亮。
- 移动端目标玩家优先布局。

不负责：

- 不改 `PlaybackState` 推导逻辑。
- 不改牌张细节样式、红五、立直牌等组件内部样式。
- 不改播放按钮、时间线行为。
- 不改 API 或 normalization。

## Desktop Layout Requirements

桌面端使用清晰四方位结构：

- 上方：对家区域，包含玩家名、分数、牌河。
- 左侧：上家区域，包含玩家名、分数、牌河。
- 右侧：下家区域，包含玩家名、分数、牌河。
- 下方：目标玩家区域，包含玩家名、分数、完整手牌、目标玩家牌河、播放控制入口。
- 中央：局信息和当前事件摘要。

布局建议：

- 牌桌容器使用固定比例或稳定 min-height，避免事件变化导致牌桌跳动。
- 玩家区块不能因为玩家名过长挤压牌张。
- 中央信息区不应遮挡牌河。
- 目标玩家区域用视觉强调，但不要用过重的装饰。

## Mobile Layout Requirements

移动端不强行显示完整四方牌桌。推荐顺序：

1. 当前局信息和当前事件。
2. 目标玩家手牌。
3. 目标玩家牌河。
4. 播放控制。
5. 其他三家压缩为可扫描的牌河/分数摘要。

移动端必须满足：

- 牌张不重叠。
- 玩家名不覆盖按钮。
- 控制按钮始终可点。
- 目标玩家手牌至少完整显示一行或可横向滚动。

## Component Expectations

可以从 `home-client.tsx` 拆出组件，例如：

- `PaipuTable`
- `SeatArea`
- `CenterStatus`
- `TargetPlayerArea`

组件命名可按现有风格调整，但要避免继续让 `home-client.tsx` 膨胀。

## Tests And Checks

运行：

```powershell
npm.cmd run lint
npm.cmd run build
```

浏览器检查：

- 桌面宽度下四名玩家都可见。
- 移动宽度下目标玩家区域完整可用。
- 切换不同局时牌桌尺寸不剧烈跳动。
- `?debug=1` 下牌桌仍是页面主视觉。

## Acceptance Criteria

- 首屏能看懂四名玩家的相对座位。
- 当前事件和目标玩家手牌在视觉上比 debug/摘要更突出。
- 牌桌区域没有明显文字遮挡、牌张覆盖、按钮压住牌张的问题。
