# Tile And River Agent

## Goal

完善牌张、牌河、副露显示，让信息更直观且不撑破布局。重点是稳定尺寸、可读性、红五/字牌/立直/摸切/副露来源的视觉区分。

## Ownership

负责：

- 牌张组件。
- 牌河组件。
- 副露组件。
- 牌张视觉状态：红五、字牌、立直宣言、摸切、当前牌。

不负责：

- 不改播放状态推导。
- 不改牌桌整体四方位布局。
- 不改播放控制行为。
- 不改 API 和 normalization。

## Tile Requirements

牌张组件应支持：

- 固定宽高，不能因文字变化撑开。
- 普通牌、紧凑牌两种尺寸。
- 万/筒/索/字牌不同颜色，但整体不要变成单一花哨配色。
- 红五 `0m`、`0p`、`0s` 有明显红五标记。
- 字牌显示更友好：可以继续显示 `1z` 这类内部表示，也可以映射为东南西北白发中；若映射，必须全站一致。
- 当前摸牌或当前切牌可高亮。

## River Requirements

牌河组件应支持：

- 每家牌河按事件进度显示。
- 长牌河自动换行或在固定区域内滚动，不撑破牌桌。
- 立直宣言牌有明确样式。
- 摸切牌可用轻量标记显示，但不要让牌面难读。
- 空牌河显示低对比占位，不占用过多空间。

## Call Requirements

副露组件应支持：

- 碰、吃、明杠、暗杠、加杠显示不同文本或轻量标记。
- 保留 tiles 顺序。
- 如果 `froms` 可用，显示来源方向；来源无法可靠显示时不要猜。
- 多个副露不能撑破玩家区域，超出时折行或滚动。

## Suggested Components

- `Tile`
- `TileRiver`
- `CallMeld`
- `SeatDiscards`

这些组件应尽量只接收展示数据，不自己推导事件历史。

## Tests And Checks

运行：

```powershell
npm.cmd run lint
npm.cmd run build
```

浏览器检查：

- 东1局推进到第 2 事件，目标牌河出现 `2z`。
- 推进到有副露事件后，副露区域出现对应吃/碰/杠。
- 长牌河不会覆盖玩家名或控制按钮。
- 移动端牌张不会横向撑破页面。

## Acceptance Criteria

- 所有牌张尺寸稳定。
- 红五、字牌、立直牌、当前动作牌至少有一种可辨识视觉差异。
- 牌河和副露信息完整可读，不再“显示不全但不知道还有内容”。
